# GIOTTO｜#034 — the Missing Piece Seeker
## 文化人類学グラレコフォーマット / QU_TED Project メンバー引き継ぎ書

---

## このアプリは何をするのか

論文を「全部読んでから考える」のではなく、**「主張（Zone C）を先に立てて、必要なものだけ読む」**ための思考フォーマット。

文化人類学のゼミ生が発表準備をするとき、最初に直面する壁——「論文200本読んだけど何が言いたいかわからない」——を解決するために作る。

## 5ゾーンの構造

| ゾーン | 英語名 | 役割 | 埋める順番 |
|---|---|---|---|
| **A** | What | 現象・対象のイメージ（イラスト・写真・キーワード） | 4番目 |
| **B** | Who said what | 主要学説 × 5（対立軸・時系列で整理） | 3番目 |
| **C** | So what / New claim | **自分の主張（仮説）** | **1番目・最重要** |
| **D** | Evidence | FWの証拠・観察・引用 | 2番目 |
| **E** | Gap / Next | 薄いところ・次に読む論文 | 5番目 |

**Zone Cを最初に書く。** これが設計の核心。

---

## Zone C（主張）の3つの型

### 型1：再解釈型（Reinterpretation）
```
「〇〇は従来△△として理解されてきたが、
 □□という視点から見ると、××として再解釈できる」
```

### 型2：反証型（Counter-evidence）
```
「〇〇学説は△△地域・時代では成立する。
 しかし□□という条件下では逆の現象が観察される」
```

### 型3：概念拡張型（Concept extension）
```
「△△という概念はもともと〇〇の文脈で使われてきた。
 しかし□□という現象を分析するときにも有効である」
```

## アプリの動作フロー

```
1. 起動
   ↓
2. 「あなたのテーマを研究している人たちが
    見落としていると思うことは何ですか？」
   という一問を表示
   ↓
3. ユーザーが答えを入力
   ↓
4. Claude APIがZone C（主張の仮説）を3つの型で提案
   ↓
5. ユーザーが選ぶ or 編集
   ↓
6. 5ゾーンのフォーマットが展開
   （BとDに何を入れるべきかもClaudeがサジェスト）
   ↓
7. Zone Eで「次に読む論文のキーワード」を提示
```

## 技術仕様（QU_TED共通ルールに従う）

- シングルHTMLファイル（Vanilla JS）
- AI：Anthropic Claude API（`claude-sonnet-4-20250514`）
- 外部ライブラリ：Google Fonts のみ
- ホスティング：GitHub Pages対応
- データ保存：なし（セッション完結型）
- APIキー：localStorage保存（他のメンバーと同じ方式）

```css
--bg: #f0ede6;       /* 象牙 */
```

- Zone Eは中列下段、左ボーダーに黒線
